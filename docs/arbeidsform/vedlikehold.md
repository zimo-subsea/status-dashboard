# Hvordan siden vedlikeholdes

## Standard

Når prosjektet endrer seg:

1. Oppdater Zimo-wikien i Obsidian hvis det er varig kunnskap.
2. Oppdater denne statiske siden hvis det påvirker hva Mads og Sverre bør gjøre nå, pilotkunder, funding, beslutninger eller blokkere.
3. Push til GitHub.
4. GitHub Actions bygger og publiserer siden med GitHub Pages.

## Hva siden skal være

- Kort intern styringsflate.
- Nå-status for Mads og Sverre.
- Neste handlinger, beslutninger, risiko og blokkere.
- Lett å lese raskt før en arbeidssesjon.

## Hva som ikke skal inn her

- Private rånotater.
- Lange møtereferater.
- Sensitive detaljer.
- Markedsføringstekst.
- Investor- eller kundepresentasjon.
- For mye teknisk rationale.

Denne siden skal være styringsflaten, ikke hele hukommelsen.
