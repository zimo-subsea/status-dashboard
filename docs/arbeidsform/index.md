# Arbeidsform

## Prinsipp

Dette er et internt arbeidsdashboard for Mads og Sverre. Det skal ikke være en presentasjonsside, investorflate eller ekstern prosjektrapport.

Det viktigste er oversikt: hva vi gjør nå, hva som er bestemt, hva som venter, og hva som blokkerer oss.

## Rollefordeling

| Person | Rolle |
| --- | --- |
| Mads | Leder prosjektet, både business og engineering. |
| Sverre | Prosjektpartner, særlig engineering. |
| Codex | Prosjektmanager og wiki-/site-vedlikeholder. |

## Arbeidsregel

- Obsidian er privat kunnskapsbase.
- Denne MkDocs-siden er den interne prosjektflaten.
- GitHub er publiseringskanalen.
- Oppgaver skal være korte og praktiske.
- Detaljert kontekst skal ligge i wikien.
- Ikke skriv for kunder, investorer eller offentligheten med mindre det er eksplisitt behov.
- Bruk tabeller, lister og korte statusfelt fremfor lange forklaringer.
